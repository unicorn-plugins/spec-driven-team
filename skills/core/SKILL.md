# Core

[CORE 활성화]

## 목표

spec-driven-team 플러그인의 핵심 행동 규범을 정의하고,
모든 스킬과 에이전트가 준수해야 할 원칙을 제시함.

## 핵심 원칙

### 1. 명세 = 코드의 단일 진실 원천 (Single Source of Truth)

명세와 코드는 항상 동기화되어야 하며, 불일치가 발견되면 즉시 해결함.

### 2. 느슨한 동기화 (Loose Coupling)

완벽한 일치를 강제하지 않음:
- 일시적 불일치 허용
- 주기적 동기화 (매일 체크)
- 사용자 선택 존중 (자동/수동/나중에)

### 3. 선언적 로직 우선

복잡한 알고리즘보다 선언적 로직을 먼저 명세화:
- CRUD 연산
- 데이터 변환/매핑
- 규칙 기반 검증

### 4. 개발자 자율성 존중

자동화는 보조 수단이며, 최종 결정은 개발자가 함:
- TODO 주석 생성 (복잡한 로직)
- 사용자 승인 (명세/코드 업데이트 전)
- 수동 개입 옵션 제공

## 행동 규범

### 스킬 행동 규범

| 스킬 | 역할 | 원칙 |
|------|------|------|
| analyze | 코드 분석 | 읽기 전용, 분류만 수행 |
| generate | 명세 생성 | 코드 수정 금지 |
| sync | 양방향 동기화 | 사용자 승인 필수 |
| watch | 상태 모니터링 | 폴링 방식 (5초 간격) |
| verify | 품질 검증 | 회귀 테스트 자동 실행 |

### 에이전트 행동 규범

| 에이전트 | 역할 | 제약 |
|----------|------|------|
| analyzer | 분석 + 권고 | file_write 금지 |
| spec-manager | 명세 관리 | 코드 수정 금지 |
| code-generator | 코드 생성 | 명세 수정 금지 |
| quality-guardian | 모니터링 + 검증 | file_write 금지 (상태 파일 제외) |

## 라우팅 규칙

모호한 요청이 들어오면 적절한 스킬로 라우팅:

| 사용자 요청 | 라우팅 대상 |
|-------------|-------------|
| "코드 분석해줘" | analyze |
| "명세 생성해줘" | generate |
| "동기화해줘" | sync |
| "상태 확인해줘" | watch |
| "테스트해줘" | verify |
| "설정해줘" | setup |
| "도움말" | help |

## MUST 규칙

- [ ] 명세-코드 불일치 시 즉시 사용자에게 알림
- [ ] 코드 생성/수정 전 반드시 사용자 승인
- [ ] 복잡한 로직은 TODO 주석으로 대체
- [ ] 모든 변경 사항은 백업 생성
- [ ] 회귀 테스트 자동 실행

## MUST NOT 규칙

- [ ] 사용자 승인 없이 코드 수정 금지
- [ ] 명세 없이 코드 생성 금지
- [ ] 복잡한 로직 자동 구현 금지
- [ ] 테스트 없이 배포 금지

## 검증 체크리스트

- [ ] 모든 스킬이 원칙을 준수하는가
- [ ] 사용자 승인 단계가 존재하는가
- [ ] 백업이 정상 생성되는가
- [ ] 회귀 테스트가 자동 실행되는가
